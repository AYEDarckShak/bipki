# Инструкция по деплою на Vercel через GitHub

Это приложение было подготовлено для деплоя на платформу **Vercel**.

## Шаг 1: Создание репозитория на GitHub

1.  Создайте новый пустой репозиторий на вашем аккаунте GitHub.
2.  Инициализируйте локальный репозиторий в папке с проектом:
    ```bash
    git init
    git add .
    git commit -m "Initial commit for Vercel deployment"
    ```
3.  Свяжите локальный репозиторий с удаленным на GitHub:
    ```bash
    git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
    git branch -M main
    git push -u origin main
    ```

## Шаг 2: Подключение к Vercel

1.  Зайдите на сайт [vercel.com](https://vercel.com/) и войдите через GitHub.
2.  Нажмите кнопку **"Add New..."** -> **"Project"**.
3.  В списке репозиториев выберите созданный вами репозиторий и нажмите **"Import"**.
4.  Vercel автоматически определит настройки (Node.js). Нажмите **"Deploy"**.

## Важное замечание о базе данных (SQLite)

Это приложение использует базу данных **SQLite** (`startup_manager.db`). На Vercel (в серверлесс-среде) файловая система является временной и доступна только для чтения после сборки.

*   **Что это значит:** Данные, которые вы добавите через интерфейс приложения на Vercel, будут стерты при перезагрузке сервера (холодном старте функции).
*   **Решение:** Для полноценной работы в продакшене рекомендуется заменить SQLite на облачную базу данных, например:
    *   **Vercel Postgres** (бесплатно в рамках Vercel)
    *   **Supabase**
    *   **MongoDB Atlas**

Для демонстрационных целей приложение будет работать с начальными данными, которые уже есть в файле `startup_manager.db`.
